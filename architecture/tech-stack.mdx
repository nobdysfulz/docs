## Frontend

| Technology | Version | Purpose |
|-----------|---------|---------|
| **React** | 18 | UI component library |
| **Vite** | 6 | Build tool and dev server |
| **React Router** | 7 | Client-side routing |
| **TanStack React Query** | 5 | Server state management and caching |
| **Tailwind CSS** | 3 | Utility-first CSS framework |
| **shadcn/ui** | Latest | Pre-built accessible UI components (Button, Card, Dialog, Tabs, etc.) |
| **Lucide React** | Latest | Icon library |
| **Recharts** | Latest | Charting library for analytics dashboards |
| **jsPDF** | Latest | Client-side PDF generation |
| **date-fns** | Latest | Date utility library |

## UI Component Library (shadcn/ui)

The application uses shadcn/ui components extensively. Key components:

| Component | Usage |
|-----------|-------|
| `Button` | All interactive actions |
| `Card` | Content containers and KPI displays |
| `Dialog` / `AlertDialog` | Modal windows |
| `Tabs` | Internal tab navigation |
| `Select` | Dropdown selections |
| `Input` / `Textarea` | Form fields |
| `Badge` | Status indicators and labels |
| `Tooltip` | Contextual help text |
| `Progress` | Progress bars for goals and usage |
| `Skeleton` | Loading state placeholders |
| `ScrollArea` | Scrollable containers |
| `Separator` | Visual dividers |
| `Sheet` | Slide-in panels |
| `Table` | Data tables |

Components are located in `src/components/ui/` and follow the shadcn/ui conventions using `class-variance-authority` for variant management and `cn()` for Tailwind class merging.

## Build & Development

### Development Server
```bash
npm run dev    # Starts Vite dev server with HMR
```

### Production Build
```bash
npm run build  # Vite production build with tree-shaking and minification
```
